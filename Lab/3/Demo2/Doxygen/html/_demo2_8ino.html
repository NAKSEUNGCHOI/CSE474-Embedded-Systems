Detailed Description

Task1 and Task2 concurrently using Synchronized Round Robin(SRRI) scheduler.

Author
Nakseung Choi 1572578 Kejin Li 1978130
Date
20-May-2022

Detailed Description: In setup function, LED, speaker, interrupt functions are called for initial setup. Task1() and Task2() are stored in void function pointer that points to an array called runningTasks[NTasks]. The rest of the space in runningTasks is stored with schedule_sync. As it goes through loop() function, (*runningTasks[i])() calls each Task 1 and 2, and both of them start to run concurrently. Sleep function, which takes a number of sleep time is employed to make a task go sleep for a certain period of time. In schedule sync function, the amount of the time that a task sleeps is synchronized by sleepTasks decrementing by -2. At the same time, timeTasks increments by 2 (meaning 2 ms) because interrupt is set to occur every 2 ms.
